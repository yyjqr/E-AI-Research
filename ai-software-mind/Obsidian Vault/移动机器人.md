YOLOv8 detection node (Python outline)

Use Ultraytics YOLOv8 model (yolov8m / yolov8m-seg) to run inference on frames.
Publish detection results as vision_msgs/Detection2DArray (standard) or a custom message containing bounding boxes, class, score and mask (if seg used).
Compute a nearest-obstacle point per frame:
If you have depth source: use depth to get 3D point under the bounding box (e.g., bottom-center).
If no depth: estimate depth from bbox size using a heuristic (calibrate distance vs. bbox height) or run a monocular depth network (MiDaS or FastDepth).
Publish a small sensor_msgs/PointCloud2 with those points for Nav2.
Example detection node (simplified, uses OpenCV + ultralytics; heuristic depth fallback):

python

#!/usr/bin/env python3
import rclpy
from rclpy.node import Node
from sensor_msgs.msg import Image, PointCloud2, PointField
from vision_msgs.msg import Detection2DArray, Detection2D, BoundingBox2D
from cv_bridge import CvBridge
from ultralytics import YOLO
import numpy as np
import struct
import time

class YoloNode(Node):
    def __init__(self):
        super().__init__('yolo_node')
        self.br = CvBridge()
        self.pub_det = self.create_publisher(Detection2DArray, 'yolo/detections', 10)
        self.pub_cloud = self.create_publisher(PointCloud2, 'yolo/nearest_obstacle', 10)
        self.subscription = self.create_subscription(Image, '/camera/image_raw', self.image_cb, 1)
        self.model = YOLO('yolov8m.pt')  # or yolov8m-seg.pt
        # heuristic params for depth estimate (example)
        self.calib_scale = 1.0
        self.get_logger().info("YOLO node started")

    def image_cb(self, msg: Image):
        cv_img = self.br.imgmsg_to_cv2(msg, desired_encoding='bgr8')
        results = self.model(cv_img)[0]  # single-frame
        det_array = Detection2DArray()
        det_array.header.stamp = self.get_clock().now().to_msg()
        det_array.header.frame_id = msg.header.frame_id or 'camera_link'
        points = []

        for r in results.boxes:
            x1, y1, x2, y2 = map(float, r.xyxy[0])
            score = float(r.conf[0])
            cls = int(r.cls[0])
            w = x2 - x1
            h = y2 - y1
            # build detection message
            det = Detection2D()
            det.header = det_array.header
            bbox = BoundingBox2D()
            bbox.center.x = (x1 + x2) / 2.0
            bbox.center.y = (y1 + y2) / 2.0
            bbox.size_x = w
            bbox.size_y = h
            det.bbox = bbox
            # add additional info in results? (replace with correct msg fields as needed)
            det_array.detections.append(det)

            # Estimate depth by heuristic: depth = k / bbox_height
            # Calibrate k for your camera & target typical sizes
            depth = self.calib_scale * (1.0 / (h + 1e-6))  # simple example
            # compute pixel coordinate bottom-center
            u = (x1 + x2) / 2.0
            v = y2
            # convert pixel to camera coordinates (requires intrinsics if available)
            # here we use a simple pinhole model fallback:
            fx = fy = 600.0  # replace with actual focal length px
            cx = cv_img.shape[1] / 2.0
            cy = cv_img.shape[0] / 2.0
            X = (u - cx) * depth / fx
            Y = (v - cy) * depth / fy
            Z = depth
            points.append((X, Y, Z))

        self.pub_det.publish(det_array)
        if points:
            pc2 = self.make_pointcloud2(points, det_array.header)
            self.pub_cloud.publish(pc2)

    def make_pointcloud2(self, points, header):
        # pack floats into binary as x,y,z fields per point
        data = b''.join(struct.pack('fff', *p) for p in points)
        pc2 = PointCloud2()
        pc2.header = header
        pc2.height = 1
        pc2.width = len(points)
        pc2.is_dense = True
        pc2.is_bigendian = False
        pc2.fields = [
            PointField(name='x', offset=0, datatype=PointField.FLOAT32, count=1),
            PointField(name='y', offset=4, datatype=PointField.FLOAT32, count=1),
            PointField(name='z', offset=8, datatype=PointField.FLOAT32, count=1),
        ]
        pc2.point_step = 12
        pc2.row_step = pc2.point_step * pc2.width
        pc2.data = data
        return pc2

def main():
    rclpy.init()
    node = YoloNode()
    rclpy.spin(node)
    node.destroy_node()
    rclpy.shutdown()
Notes:

Replace heuristic depth with a proper depth network (recommended) or a real depth sensor for reliable Nav2 obstacle avoidance.
Use segmentation masks from yolov8m-seg if you want to generate dense obstacle point clouds by sampling mask pixels and projecting them into 3D (requires depth or monocular depth).
Nav2 integration (how to feed obstacles into costmap)
Nav2 costmap can consume sensor_msgs/PointCloud2 by using an obstacle layer or voxel layer configured to read from the topic you publish.
Example costmap YAML snippet (costmap_common_params.yaml):
yaml

obstacle_layer:
  enabled: true
  observation_sources: yolo_points
  yolo_points: {sensor_frame: camera_link, data_type: PointCloud2, topic: /yolo/nearest_obstacle, marking: true, clearing: true}
Put this under your local/global costmap configuration and tune inflation_layer / obstacle_range / raytrace_range.
Important: Ensure frame transforms exist between camera frame and the robot base frame (use static_transform_publisher or tf2).
Arduino UNO integration (drive commands)
Arduino UNO is limited; recommended approaches:
Use a serial bridge node on Jetson (Python) that writes simple commands (e.g., "FWD:0.5\n") to the Arduino over USB Serial. Arduino runs a small sketch parsing commands to control motors.
Alternatively use rosserial or micro-ROS. micro-ROS on UNO is constrained; rosserial might be easier.
Example Jetson-side simple serial sender (Python):
python

import serial
import rclpy
from geometry_msgs.msg import Twist

ser = serial.Serial('/dev/ttyUSB0', 115200, timeout=0.1)

def twist_cb(msg: Twist):
    # encode linear.x and angular.z into a small text frame
    cmd = f"V:{msg.linear.x:.2f},W:{msg.angular.z:.2f}\n"
    ser.write(cmd.encode('ascii'))
On Arduino, parse the line and set PWM / motor directions.
Performance tips for Jetson Orin Nano
Use TensorRT-optimized YOLOv8 model if possible (Ultralytics supports exporting to ONNX/TensorRT).
Lower model size (yolov8n or yolov8s) if you need higher fps; yolov8m likely ok but test.
Use GStreamer with nvarguscamerasrc where applicable for UVC or CSI cameras for low-latency.
Run inference in a separate thread and use a bounded queue for frames to avoid growing latency.
Tuning & calibration
Calibrate camera intrinsics (focal length fx, fy and principal point cx, cy). Accurate intrinsics let you project pixel -> camera frame if depth available.
If using monocular depth, calibrate scale with a small dataset of known distances.
Tune Nav2 costmap parameters: inflation radius, obstacle range, observation sources update frequency.
